## How to Build, Expand, and Maintain the Lower Limb Dataset

## Dataset Structure

```
dataset/lower_limb/
├── raw/                        ← Place your raw videos here (NOT in git)
│   ├── ankle/
│   ├── calf/
│   ├── hamstring/
│   ├── heel_slide/
│   ├── hip_abduction/
│   ├── knee_extension/
│   ├── leg_raise/
│   ├── quadriceps_set/
│   └── toe_raise/
│
├── processed/                  ← Generated by the pipeline (NOT in git)
│   ├── frame_csv/
│   │   └── lower_limb_frame_labels.csv
│   ├── tensors/
│   │   ├── ankle_video_01.npy
│   │   └── …
│   ├── train/
│   │   └── train_labels.csv
│   ├── val/
│   │   └── val_labels.csv
│   └── test/
│       └── test_labels.csv
│
└── metadata/                   ← Tracked in git ✅
    ├── class_map.json           ← Exercise class → integer ID mapping
    ├── dataset_statistics.json  ← Per-class sample counts
    ├── split_manifest.json      ← Complete record of every sample + split
    └── extraction_report.json   ← SHA-256 of every processed video
```

## Adding New Videos

### Method A — Organized Videos (recommended)

Place videos directly into the appropriate class subfolder:
```bash
cp my_new_ankle_video.mp4 dataset/lower_limb/raw/ankle/
```

Then run the pipeline:
```bash
python automation/run_lower_pipeline.py
```

Only the new video is processed. Everything else is unchanged.

### Method B — Unorganized Videos (bulk import)

If you have a folder of mixed videos with no class subfolders:
```bash
# 1. Preview keyword classification
python tools/classify_new_videos.py --source /path/to/mixed/videos --dry-run

# 2. Review and edit tools/proposed_classification.json
# 3. Confirm and copy
python tools/classify_new_videos.py --confirm
```

## Supported Video Formats

`.mp4`, `.avi`, `.mov`, `.mkv`, `.wmv`, `.flv`, `.webm`

**Recommended format:** `.mp4` (H.264), 720p–1080p, 24–60 fps.

---

## Video Naming Convention

Video filenames become the `sample_name` in the dataset CSVs. The pipeline infers the class from the parent folder name, not the filename.

Good:
```
ankle/Ankle_Pumps_01.mp4
ankle/Ankle_Dorsiflexion_Band.mp4
```

Also fine (class comes from folder, not filename):
```
ankle/video_001.mp4
```

## Class Map

| Class Name | Label ID | Example Exercises |
|---|---|---|
| `ankle` | 0 | Ankle pumps, dorsiflexion, plantar flexion |
| `calf` | 1 | Calf raises, eccentric heel drops |
| `hamstring` | 2 | Hamstring stretches, curls |
| `heel_slide` | 3 | Supine heel slides |
| `hip_abduction` | 4 | Hip bridge, hip abduction, clam shells |
| `knee_extension` | 5 | Terminal knee extension, leg press |
| `leg_raise` | 6 | Straight leg raise, hip flexion |
| `quadriceps_set` | 7 | Quad sets, standing quad stretch |
| `toe_raise` | 8 | Toe raises, tibialis anterior |

## Tensor Format

Each video is converted to a NumPy array of shape `(4, 300, 10, 1)`:

| Dimension | Size | Meaning |
|---|---|---|
| C | 4 | x, y, z, visibility |
| T | 300 | Temporal frames (resampled) |
| V | 10 | Joints (MediaPipe landmarks 23–32) |
| M | 1 | Persons per frame |

## Dataset Versioning

Each time the pipeline runs with new videos, a versioned snapshot is created:

```bash
# List all versions
python tools/version_dataset.py --list

# Manually create a new version
python tools/version_dataset.py --domain lower_limb
```

Versions are stored as lightweight JSON manifests in `dataset_versions/` — not the raw videos or tensors.

---

## Verifying Dataset Integrity

```bash
python src/preprocessing/validate_dataset.py
```

This runs 8 checks:
1. Raw directories exist
2. Frame CSV is valid
3. Corrupted videos
4. Cross-class duplicate detection
5. Tensor shape validation
6. Split CSV consistency
7. Class balance check
8. Class coverage check

## Downloading the Dataset (for reproducibility)

Raw videos are not stored in git. To reproduce results:

1. Obtain the raw video collection (contact the authors).
2. Place videos into `dataset/lower_limb/raw/<class>/`.
3. Verify against the published manifest:
   ```bash
   python tools/version_dataset.py --list
   cat dataset_versions/lower_limb_v1/manifest.json | python -m json.tool | head -40
   ```
4. Run the pipeline:
   ```bash
   python automation/run_lower_pipeline.py
   ```
